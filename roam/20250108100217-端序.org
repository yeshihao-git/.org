#+title: 字节序
#+filetags: network

* 字节序
- 字节序 :: 是 *数据存储的顺序* ，分为：
  1. [[id:3b86edec-8a79-48e7-af6f-9b43052781b1][大端序]]
  2. [[id:6c58d2a2-2c87-413a-90b3-c8313f538c5f][小端序]]

*网络通信时需要进行 端序处理*
#+name: why？
#+begin_comment
计算机内部、网络对字节的存储顺序不同，因此需要在网络通信时进行端序处理
#+end_comment
- 网络传输  ：一般为大端序
- 处理器体系：大端序、小端序都有

#+name: 助记
#+begin_comment
小主机、大网络
#+end_comment


** 大端序
:PROPERTIES:
:ID:       3b86edec-8a79-48e7-af6f-9b43052781b1
:END:
- 大端序 :: 高字节 存储在 低地址，低字节 存储在 高地址，一般作为 *网络字节序*


** 小端序
:PROPERTIES:
:ID:       6c58d2a2-2c87-413a-90b3-c8313f538c5f
:END:
- 小端序 :: 低字节 存储在 低地址，高字节 存储在 高地址，一般作为 *主机字节序*
